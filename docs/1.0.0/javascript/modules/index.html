Modules

  • Overview
  • Usage
  • Creating modules

Overview

Hubble's modules are unique and manager by a module loader. The module load is the only variable available in the global namespace. All other components, variables and objects are held within the module loader. It works in a similar way to an IOC container.

Usage

To set a key pair via the module loader use the Modules.set(); method.

You can also set objects and arrays via the module loader.

You can also set a class object as a key pair. This is useful for a utility module that gets reused multiple times.

When you set a class using set();, the class is NOT invoked by the container until you retrieve it via the key. You will ALWAYS receive a new instance of the class object.

The singleton(); method ensures you always get the same global instance of a class object back. This is useful for a global module that needs to be accessed by multiple times from different environments.

When you set a class using singleton, the class is NOT invoked by the container until the first time you retrieve it via the key. You will ALWAYS receive a the same instance of the class object.

The require(); method returns variable, object, array, class, or class instance by key.

You can pass arguments to the object constructor as well.

You can also use the get(); method which is just an alias for require();.

If you want a module to instantiate straight away, chain the set and get methods together.

The all(); method returns an object of all the key pairs in the container.

The keys(); method returns an array of all the keys in the container.

The has(); method checks if a value exists by key.

The remove(); method removes a key/pair from the container.

If the value is a class object and has a destruct(); method, the container will call destruct(); before it destroys the keypair. This is useful if a a module has eventListeners bound to DOM objects as this is where they should be unbound.

The clear(); method removes all key/pairs from the container.

The refresh(); method removes all key/pairs from the container and reinitializes them. This is useful for large DOM changes where you want to remove unused eventListeners.

Creating modules

To create your own module, you should follow a specific class structure.

  • Wrap the class in closure ((function() { ... }());) so it is not available via the global namespace.
  • Have a publicly callable destruct(); method that unbinds any DOM eventListeners
  • Private methods should all have _ infront of the method name e.g. _privateMethod();
  • Use the object prototype where possible.
